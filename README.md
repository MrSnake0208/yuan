# astrbot-plugin-yuan-redeem

一个用于 AstrBot 的代号鸢兑换插件。

## 功能

- 用户私聊机器人后，可通过分步对话绑定自己的“角色ID”和“角色名”
- 每个用户与机器人的绑定信息独立保存，互不干扰
- 管理员可维护全局兑换码队列
- 用户发送 `代号鸢兑换` 后，机器人会按顺序帮其兑换所有尚未自动跳过的全局兑换码
- 插件会按兑换结果分类记录：明确成功会跳过，未知结果会冻结等待补规则，请求异常可后续重试

## 用户指令

以下指令建议在**私聊**中使用，并通过 AstrBot 配置的命令前缀触发（例如 `#`）：

- `#绑定代号鸢`：启动绑定流程
- `#解绑代号鸢`：解绑当前账号
- `#代号鸢绑定状态`：查看当前绑定信息
- `#代号鸢兑换`：兑换所有尚未自动跳过的全局兑换码
- `#查询代号鸢绑定`、`#查看代号鸢绑定`：`#代号鸢绑定状态` 的别名

绑定流程中发送 `取消` 或 `退出` 可终止本次绑定。

## 管理员指令

- 管理员指令同样通过 AstrBot 配置的命令前缀触发，且仅支持在私聊中使用
- `#添加代号鸢兑换码 <code...>`：添加一个或多个全局兑换码，支持空格、逗号分隔
- `#新增代号鸢兑换码 <code...>`：同上
- `#删除代号鸢兑换码 <code>`：删除指定兑换码
- `#查看代号鸢兑换码`：查看当前全局兑换码列表
- `#代号鸢兑换码列表`：`#查看代号鸢兑换码` 的别名
- `#清空代号鸢兑换码`：清空当前全部全局兑换码
- 示例：`#添加代号鸢兑换码 CODE123`、`#查看代号鸢兑换码`

管理员权限通过 AstrBot 的 `ADMIN` 权限过滤器判定。

## 存储说明

插件会在同目录生成 `yuan_redeem.sqlite3`，用于保存：

- 用户绑定信息
- 全局兑换码
- 每个用户的兑换记录

## 兑换接口

插件会向以下接口发送 `POST` 请求：

- `https://p11132-game-adapter.qookkagames.com/cms/active_code/change`

请求体示例：

```json
{
  "player_name": "玩家名",
  "player_id": "123456",
  "code": "ACTIVECODE"
}
```

## 开发说明

当前仓库是基于 AstrBot 插件模板改造的单文件实现，主逻辑位于 `main.py`。

当前成功/最终失败的消息规则预留在 `main.py` 的 `SUCCESS_MESSAGE_RULES` 与 `FINAL_FAILURE_MESSAGE_RULES`，拿到真实接口文案后可直接补充。
